# Task 1: Using Ping and Traceroute

## Objective
Learn how to use `ping` to test network connectivity and `traceroute` to trace the path of packets to a destination.

## Exercises
1. Use the `ping` command to check if you can reach `google.com`.
   ```bash
   ping google.com
   ```
2. Ping a host that doesn't exist to see what happens.
   ```bash
   ping a-host-that-does-not-exist.com
   ```
3. Use `traceroute` (or `tracert` on Windows) to see the path that packets take to get to `google.com`.
   ```bash
   traceroute google.com
   ```
4. What is the difference between `ping` and `traceroute`?
5. When would you use one over the other?
